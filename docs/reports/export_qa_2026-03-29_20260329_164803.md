## Code Fixes Applied
1. `scripts/export_building_fbx.py`
- Added GLB sidecar export (`<address>.glb`) by default.
- Added CLI flag `--skip-glb`.
- Metadata now records `glb_path`.

2. `scripts/bake_utils.py`
- Added `export_glb(...)` helper.
- Extended `write_export_metadata(...)` with optional `glb_path`.

3. `scripts/validate_export_pipeline.py`
- Added FBX-load fallback to sibling GLB when trimesh cannot load FBX.
- Added scene-to-mesh coercion for GLB loads that return `trimesh.Scene`.
- Excluded `_LOD*` and `_collision` files from top-level scan to prevent false standalone failures.

4. `scripts/generate_collision_mesh.py`
- Blender 5 operator compatibility fix (`temp_override` for modifier apply).
- FBX export compatibility fallback when `use_default_deform` unsupported.

5. `scripts/generate_lods.py`
- Blender 5 operator compatibility fix (`temp_override`).
- FBX export compatibility fallback when `use_default_deform` unsupported.
- Removed premature `clear_scene()` calls that wiped source geometry.
- Fixed LOD2 object-removal bug (`StructRNA ... removed`) by caching name before delete.

6. `scripts/build_unity_prefab_manifest.py` and `scripts/build_unreal_datasmith.py`
- Recursive FBX scanning (`rglob`) across `outputs/exports/**`.
- LOD/collision companion detection fixed for nested per-address directories and uppercase `_LOD*` naming.
